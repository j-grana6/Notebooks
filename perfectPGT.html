A Game of perfect information¶

In [2]:
Image('perfectinfo.png')
Out[2]:
  • This game has 72 Nash equilibria
  • Only 1 of the equilibria is "subgame perfect"
  • However, we know the MCTS and CRM for multiplayers have no theoretical guarantees to converge to the subgame perfect equilibrium.
  • For reference, the subgame perfect strategy for player 1 is (A if 1, B if 2) and for player 2 is (X if (1,A), X if (1, B), X if (1, C), Y if (2,A), Y if (2,B), Z if (2,C))

PGT Intelligence¶

  • The intelligence of a strategy profile of both players $(s_1, s_2)$ is the intelligence of $s_1$ against $s_2$ times the intelligence of $s_2$ against $s_1$.

  • The intelligence of $s_1$ against $s_2$ is the fraction of player 1's strategies that do worse (lower expected utility) against $s_2$ than $s_1$.

  • All Nash equilibrium strategy profiles have an intelligence of 1

Trembling¶

  • Instead of computing the intelligence of $s_1$ against $s_2$ as the proportion of profiles that do worse than $s_1$ against $s_2$, we assume the player 2 plays $s_2$ with probability $p$ and trembles uniformly over all other strategies with probability $(1-p)$. Then we compute player 1's expected utility by averaging over player 2's trambles. The intellgience is then the proportion of player 1's stratgies that do worse than $s_1$ against the "trembling" $s_2$

  • We can show (analytically) that under any tremble probability, a strategy profile has an intelligence of 1 iff it is subgame perfect.

  • However, since intelligence ultimately determine the probability distribution voer outcomes (profiles with higher intelligence have higher probability of being played) we are interested to see the trembling intelligences of the non-subgame perfect equilibria.

def EU_with_trembles_perfect_info(p1strat, p2strat, utils=utils, p1top2map=mapdict, p1trembleprob=.2, 
                     p2trembleprob=.2, p1tremblestrats=perfect_strat_1, p2tremblestrats=perfect_strat_2):
    """ 
    Computes the expected utility of player 1 and player 2.
    
    Nature node is hard coded to p=.5 for both outcomes.
    
    Setting trembleprobs to 0 gives standard intelligence
    """
    highukey = "H"+p1strat[0]+p2strat[mapdict[p1strat[0]]]
    lowukey =  "L"+p1strat[1]+p2strat[3 + mapdict[p1strat[1]]]
    p1eunotremb = (1-p2trembleprob) * (.5 * utils[highukey][0] + .5 * utils[lowukey][0])
    p2eunotremb = (1-p1trembleprob) * (.5 * utils[highukey][1] + .5 * utils[lowukey][1])
    if p1trembleprob + p2trembleprob ==0:
        return p1eunotremb, p2eunotremb
    ### Get the utility that is associated with p2 trembling
    p1epovern = p1trembleprob/float(len(p1tremblestrats) - 1)
    p2epovern = p2trembleprob/float(len(p2tremblestrats) - 1)
    p1tremblingeu = 0 # Utility of p1 when p2 trembles
    p2tremblingeu = 0 # Utility of p2 when p1 trembles
    for s in p2tremblestrats:
        if s != p2strat:
            highukey = "H"+p1strat[0]+s[mapdict[p1strat[0]]]
            lowukey =  "L"+p1strat[1]+s[3 + mapdict[p1strat[1]]]
            p1tremblingeu += p2epovern * (.5 * utils[highukey][0] + .5 * utils[lowukey][0])
    for s in p1tremblestrats:
        if s != p1strat:
            highukey = "H"+s[0]+p2strat[mapdict[s[0]]]
            lowukey =  "L"+s[1]+p2strat[3 + mapdict[s[1]]]
            p2tremblingeu += p1epovern * (.5 * utils[highukey][1] + .5 * utils[lowukey][1])
    return p1eunotremb + p1tremblingeu, p2eunotremb + p2tremblingeu

def get_trembling_intelligence_perfect_info(profile, p1trembleprob=.2, p2trembleprob=.2):
    """
    Get's the trembling intelligence of a profile under trembling
    """
    p1strat = profile[0]
    p2strat = profile[1]
    util = EU_with_trembles_perfect_info(p1strat, p2strat, p1trembleprob=p1trembleprob, p2trembleprob=p2trembleprob)
    #print util
    p1_numalt = float(len(perfect_strat_1))
    p2_numalt = float(len(perfect_strat_2))

    num_no_betterp2 = 0
    for s in perfect_strat_1:

# Results_Class
class TremblingIntelResults(object):
    def __init__(self, res, stratprofiles = perfect_stratprofiles, perfect_strat_1=perfect_strat_1,
                 perfect_strat_2=perfect_strat_2):
        self.res = np.asarray(res)
        self.stratprofiles = stratprofiles
        self.sorted_ix = np.argsort(self.res)[::-1]
        self.perfect_strat_1 = perfect_strat_1
        self.perfect_strat_2 = perfect_strat_2
        
    def make_hist(self):
        plt.hist(self.res)
    
    def EU_perfect_info(self, p1strat, p2strat, utils=utils, p1top2map=mapdict):
        """
        No trembling EU.  USed to get Nash-eq
        """
        highukey = "H"+p1strat[0]+p2strat[mapdict[p1strat[0]]]
        lowukey =  "L"+p1strat[1]+p2strat[3 + mapdict[p1strat[1]]]
        p1eu = .5 * utils[highukey][0] + .5 * utils[lowukey][0]
        p2eu = .5 * utils[highukey][1] + .5 * utils[lowukey][1]
        return p1eu, p2eu
    
    def get_perfect_intelligence(profile):
        """
        Get's the non-trembling intelligences"""
        p1strat = profile[0]
        p2strat = profile[1]
        util = EU_perfect_info(p1strat, p2strat)
        #print util
        p1_numalt = float(len(self.perfect_strat_1))
        p2_numalt = float(len(self.perfect_strat_2))
        num_no_betterp1 = 0
        num_no_betterp2 = 0
        for s in self.perfect_strat_1:
            if EU_perfect_info(s, p2strat)[0] <= util[0]:
                num_no_betterp1 +=1
        for s in self.perfect_strat_2:
            if EU_perfect_info(p1strat, s)[1] <= util[1]:
                num_no_betterp2 +=1
        return (num_no_betterp1-1)/(p1_numalt - 1) * (num_no_betterp2 - 1)/(p2_numalt -1)
    
    def get_perfect_info_dist():
        intels_perfect = []
        for p in self.stratprofiles:
            intels_perfect.append(self.get_perfect_intelligence(p))
        return intels_perfect
    
    def eu_dist(self, power = 1, binwidth=10):
        total_prob = np.sum(self.res**power)
        probs = self.res**power/float(total_prob)
        eus = []
        for s in self.stratprofiles:
            eus.append(self.EU_perfect_info(s[0], s[1]))
        p1u = defaultdict(int)
        p2u = defaultdict(int)
        for ix, atom in enumerate(eus):
            p1u[int(atom[0]/binwidth) * binwidth ] += probs[ix]
            p2u[int(atom[1]/binwidth) * binwidth ] += probs[ix]
        return p1u, p2u
    
    def eu_hist(self, power=1, binwidth=10):
        eu_dicts = self.eu_dist(power=power, binwidth=binwidth)
        eu_subgame_perfect = [(625+250)/2. , (312. + 125.)/2.]
        eu_dicts_exact = self.eu_dist(power=power, binwidth=1)
        p1eeu = 0
        p2eeu = 0
        for k, v in eu_dicts_exact[0].iteritems():
            p1eeu += k * v
        for k, v in eu_dicts_exact[1].iteritems():
            p2eeu += k * v
        fig, [ax1, ax2] = plt.subplots(2)
        ax1.bar(eu_dicts[0].keys(), eu_dicts[0].values(), width=binwidth, color='r', alpha=.35)
        ax1.set_xlabel("Leader\'s expected utility")
        ax1.set_ylabel("Probability")
        ax1.vlines(eu_subgame_perfect[0], 0, .2)
        ax1.vlines(p1eeu, 0, .2, color='green')
        ax2.bar(eu_dicts[1].keys(), eu_dicts[1].values(), width=binwidth, color='r', alpha=.25)
        ax2.set_xlabel("Followers\'s expected utility")
        ax2.set_ylabel("Probability")
        ax2.vlines(eu_subgame_perfect[1], 0, .2)
        ax2.vlines(p2eeu, 0, .2, color='green')
        plt.tight_layout()
        fig.text(1, .5, 'The green vertical line is the expected utility of the intelligence profile. \n' + \
                        'The black line is the SPNE expected utility')
        return fig, ax1, ax2
    
    def eu_cdf(self, power=1):
        p1_dist, p2_dist = self.eu_dist( power = power)
        p1k = p1_dist.keys()
        p1v = p1_dist.values()
        p1_srtd = np.argsort(p1k)
        cumprob_p1 = np.cumsum(np.asarray(p1v)[p1_srtd])
        p2k = p2_dist.keys()
        p2v = p2_dist.values()
        p2_srtd = np.argsort(p2k)
        cumprob_p2 = np.cumsum(np.asarray(p2v)[p2_srtd])
        fig, [ax1, ax2] = plt.subplots(2)
        ax1.plot(np.asarray(p1k)[p1_srtd], cumprob_p1)
        ax1.set_xlabel("CDF of Leader\'s Utility")
        ax2.plot(np.asarray(p2k)[p2_srtd], cumprob_p2)
        ax2.set_xlabel("CDF of Followers\'s Utility")
        plt.tight_layout()
        return fig, ax1, ax2

Off path Intelligence in Games of Perfect Information¶

Here we combine the real intelligence with the full game intelligence described by James as:

  1. Calculate the intelligence of $s_L$ and $s_F$ in the usual way, i.e. on-path, and call it $IQ(i,s,on-path)$. While doing that, keep track of all unvisited information sets for each player, i.e. the set of off-path information sets, $H_i$.

  2. For each player $i$, for each unvisited information set $h$ in $H_i$, and for each state $n$ in $h$, calculate the intelligence of $s_i$ in information set $h$ given that the state is $n$, call this $IQ(i,s,n)$.

  3. For each player $i$, for each unvisited information set $h$, take the average intelligence across nodes $n$ in $h$: $IQ(i,s,h)=1/|h| * \sum_{n\in h} IQ(i,s,n)$

  4. Take the average of $IQ(i,s,h)$ across unvisited information sets in $H_i: IQ(i,s,off-path) = 1/|H_i| * sum_{h\in H_i} IQ(i,s,h).$

  5. Calculate $IQ(i,s,full-game)$ = $p*IQ(i,s,on-path) + (1-p)*IQ(i,s,off-path)$, where p is a parameter we control based on the importance of off-path play. Let’s start with something like $p=.5$.

Brief summary of both intelligence measures in games of perfect information¶

For trembling¶

  • The parameters we consider are the probability of the tremble (and the distribution over trembling actions) and the "power" that we use to raise the intelligences and put more weight on higher intelligences.
  • Variations in the tremble probability do not matter much.
  • When the strategy space is large, trembling will only be difference than regular intels when the "power" parameter is very high. Otherwise, trembling is very similar to the non-trembling intelligences.
  • Depending on the parameters, the expected utility over the PGT distribution can be greater than or less than the SPNE expected utility.

For "off-path" intelligence¶

  • The parameters we control here are again the power parameter but also the weight we give to off path intelligences
  • Varying the weight on off path intelligence can have rather significant effects on expected values.

Comparing and both¶

  • In both models, changing the parameters also changes the order of profiles when ranked in order of intelligence
  • The off-path intelligence and the trembling intelligence do not necessarily agree, and can produce different results. However, I think that in general, there will exist parameters for both that yield the same PGT distribution.
  • One thing we want to keep in mind is that we want to assign lower probabilities to profiles with larger threats. Since intelligence is ordinal in both the trembling and the off-path intelligence calculations, changes in payoffs might not necessarily lead to changes in intelligences (more on this to come).

class ImperfectResults(object):
    def __init__(self, res, stratprofiles = imperfect_stratprofiles, imperfect_strat_1 = strat_1,
                 imperfect_strat_2=strat_2):
        self.res = np.asarray(res)
        self.stratprofiles = stratprofiles
        self.sorted_ix = np.argsort(self.res)[::-1]
        self.imperfect_strat_1 = imperfect_strat_1
        self.imperfect_strat_2 = imperfect_strat_2
        
    def make_hist(self):
        plt.hist(self.res)
    
    
    def get_perfect_intelligence(self, profile):
        """
        Get's the non-trembling intelligences"""
        p1strat = profile[0]
        p2strat = profile[1]
        util = EU_perfect_info(p1strat, p2strat)
        #print util
        p1_numalt = float(len(self.perfect_strat_1))
        p2_numalt = float(len(self.perfect_strat_2))
        num_no_betterp1 = 0
        num_no_betterp2 = 0
        for s in self.perfect_strat_1:
            if EU_perfect_info(s, p2strat)[0] <= util[0]:
                num_no_betterp1 +=1
        for s in self.perfect_strat_2:
            if EU_perfect_info(p1strat, s)[1] <= util[1]:
                num_no_betterp2 +=1
        return (num_no_betterp1-1)/(p1_numalt - 1) * (num_no_betterp2 - 1)/(p2_numalt -1)
    
    
    def EU_imperfect(self, p1strat, p2strat, utils=utils, p1top2map=mapdict):
        highukey = "H"+p1strat[0]+p2strat[mapdict[p1strat[0]]]
        lowukey =  "L"+p1strat[1]+p2strat[mapdict[p1strat[1]]]
        p1eu = .5 * utils[highukey][0] + .5 * utils[lowukey][0]
        p2eu = .5 * utils[highukey][1] + .5 * utils[lowukey][1]
        return p1eu, p2eu
    
    def eu_dist(self, power = 1, binwidth=10):
        total_prob = np.sum(self.res**power)
        probs = self.res**power/float(total_prob)
        eus = []
        for s in self.stratprofiles:
            eus.append(self.EU_imperfect(s[0], s[1]))
        p1u = defaultdict(int)
        p2u = defaultdict(int)
        for ix, atom in enumerate(eus):
            p1u[int(atom[0]/binwidth) * binwidth ] += probs[ix]
            p2u[int(atom[1]/binwidth) * binwidth ] += probs[ix]
        return p1u, p2u
    
    def eu_hist(self, power=1, binwidth=10):
        eu_dicts = self.eu_dist(power=power, binwidth=binwidth)
        eu_subgame_perfect = [(625+250)/2. , (312. + 125.)/2.]
        eu_dicts_exact = self.eu_dist(power=power, binwidth=1)
        p1eeu = 0
        p2eeu = 0
        for k, v in eu_dicts_exact[0].iteritems():
            p1eeu += k * v
        for k, v in eu_dicts_exact[1].iteritems():
            p2eeu += k * v
        fig, [ax1, ax2] = plt.subplots(2)
        ax1.bar(eu_dicts[0].keys(), eu_dicts[0].values(), width=binwidth, color='r', alpha=.35)
        ax1.set_xlabel("Leader\'s expected utility")
        ax1.set_ylabel("Probability")
        ax1.vlines(eu_subgame_perfect[0], 0, .2)

        ax2.bar(eu_dicts[1].keys(), eu_dicts[1].values(), width=binwidth, color='r', alpha=.25)
        ax2.set_xlabel("Followers\'s expected utility")

        ax2.vlines(p2eeu, 0, .2, color='green')
        plt.tight_layout()
        fig.text(1, .5, 'The green vertical line is the expected utility of the intelligence profile. \n' + \
                        'The black line is the SPNE expected utility')
        return fig, ax1, ax2
    
    def eu_cdf(self, power=1):
        p1_dist, p2_dist = self.eu_dist( power = power)
        p1k = p1_dist.keys()
        p1v = p1_dist.values()
        p1_srtd = np.argsort(p1k)
        cumprob_p1 = np.cumsum(np.asarray(p1v)[p1_srtd])

        ax1.plot(np.asarray(p1k)[p1_srtd], cumprob_p1)
        ax1.set_xlabel("CDF of Leader\'s Utility")
        ax2.plot(np.asarray(p2k)[p2_srtd], cumprob_p2)
        ax2.set_xlabel("CDF of Followers\'s Utility")
        plt.tight_layout()
        return fig, ax1, ax2

def get_off_path_intel_average_over_states(profile, p=.5):

def EU_with_trembles_imperfect(p1strat, p2strat, utils=utils, p1top2map=mapdict, p1trembleprob=.2, 
                     p2trembleprob=.2, p1tremblestrats=strat_1, p2tremblestrats=strat_2):
    highukey = "H"+p1strat[0]+p2strat[mapdict[p1strat[0]]]
    lowukey =  "L"+p1strat[1]+p2strat[mapdict[p1strat[1]]]
    p1eunotremb = (1-p2trembleprob) * (.5 * utils[highukey][0] + .5 * utils[lowukey][0])
    p2eunotremb = (1-p1trembleprob) * (.5 * utils[highukey][1] + .5 * utils[lowukey][1])
    ### Get the utility that is associated with p2 trembling
    p1epovern = p1trembleprob/float(len(p1tremblestrats) - 1)
    p2epovern = p2trembleprob/float(len(p2tremblestrats) - 1)
    p1tremblingeu = 0 # Utility of p1 when p2 trembles
    p2tremblingeu = 0 # Utility of p2 when p1 trembles
    for s in p2tremblestrats:
        if s != p2strat:
            highukey = "H"+p1strat[0]+s[mapdict[p1strat[0]]]
            lowukey =  "L"+p1strat[1]+s[mapdict[p1strat[1]]]
            p1tremblingeu += p2epovern * (.5 * utils[highukey][0] + .5 * utils[lowukey][0])
    for s in p1tremblestrats:
        if s != p1strat:
            highukey = "H"+s[0]+p2strat[mapdict[s[0]]]

    return p1eunotremb + p1tremblingeu, p2eunotremb + p2tremblingeu

  • The resulting PGT distribution is not sensitive to specifications of the the parameters except the power parameter.

  • Are other quantities of interest as invariant to parameter changes?

Summary¶

We are now able to identify the subgame perfect equilibrium from the non equilibrium and non-subgame perfect profiles. However, turning this into a PGT distribution is tricky since many non-subgame perfect profiles have high intelligences.

One way around this is that the literature notes that non-credible threats become credible when the cost of the threat is small but the value of the opponent "swallowing" the threat is large. If we can show that this type of PGT distribution follows such a pattern, I think it will become a viable alternative for games of perfect information.
