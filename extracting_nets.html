The flow¶

  • 1) Randomly select a start time
  • 2) Extract nodes with high traffic (incomping and outgoing messages greater than a parameter)
  • 3) Using only the high traffic nodes, find all edges such that the number of messages is greater than some parameter
  • 4) Make the graph with nodes that meet criteria (2) and such that there is at least one edge that meets criteria 3)

Plan¶

  • For the real data section we simply randomly choose a sub-network out of the allowable networks that fit the criteria defined by the algorithm as well as some other standard criteris (i.e. minimum number of nodes/edges, connectivity etc.)

  • For the rates, I would just estimate them from data at one point in time (i.e. the message counts). Ideally, we would estimate the rates based on the entire 28 day sample but since the rates are not stationary, that adds an extra complication.

  • Another possible alternative is to use the same 30 minute window for every weekday and use the actual message times. This might work but I'm not sure if we will have enough data. However, if we cn find a two hour window where the rates are relatively constant every (week)day and we shrink the observation window to 15 minutes, we might be able to do this.
