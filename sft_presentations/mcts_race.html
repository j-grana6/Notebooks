Event Driven Hide and Seek - Exploring MCTS algorithms¶

Outline¶

In this notebook, we will construct a simple model known as "event driven hide and seek."

  • In the model, there will be an attacker and a defender on a two dimensional grid.
  • The agents face an opportunity to move according to an exogenous rate.
  • When it is a player's turn to move, they choose a direction to move.
  • The seeker receives a reward if he catches the seeker. The seeker receives a negative (discounted) reward when he is caught. To keep things simple, this is a zero sum game.
  • There is discounting
  • The game ends if the attacker catches the seeker.
  • We will model this as a "fully observable Event-Driven Game" (to be defined below). Intuitively, this means that the seeker knows the location of the hider and vice versa.
  • There is no noise element in this game.

Specifics¶

Let $W$ be an $N\times N$ grid. Let $\mathbf{L}$ be the set $\{1,2,3...N\}$

Let $V_h$ and $V_s$ be SFTs representing the hider and the seeker, respectively.

Let $S_v = (\mathbf{L}\times \mathbf{L})^2$ for $v=V_h, V_s$. Intuitively, the state of an SFT is its location on the grid and the location of the opponent.

Let $\lambda_{h}, \lambda_s$ be the rate of spontaneous events for each SFT. More formally, there is a $\lambda$ for each element of the state of the SFT. Here we assume $\lambda$ is constant across states, but not necessarily the same for the hider and seeker.

The message space $M_v$ for $v= V_h, V_s$ is also $\mathbf{L}\times \mathbf{L}$. In other words, the SFTs broadcast their location.

$\lambda_m=\infty$ for all $m\in M_v$. Messages traverse edges instantaneously.

$U_h= - e^{-\gamma t^*}$

and

$U_s= e^{-\gamma t^*}$,

where $\gamma$ is a discount rate and $t^* = \min_t(S_h(t) = S_s(t))$. There is only one reward and it occurs when the seeker finds the hider.

The agents must set $\pi_sp(\mathcal{Y}' s' |s)$ under the restriction that the only element of the state they can change is their location, their state can only change by moving at most one square and whenever they move, they transmit a message to their opponent.

This is only a rough mathematical sketch. However, we call this game "fully observable" in the sense that there exists a one-to-one mapping between the state of the agents and the state of the system as a whole. That is, an agent can observe his internal state and know exactly the state of the system.

class Player(object):
    """ 
    Either hider or seeker
    
    Tables are indexed by [hider loc, seeker_loc]
    """
    def __init__(self, rate, s0, name):
        self.rate=rate
        self.loc=s0 # Just the location
        self.name=name
        
    def create_value_and_counts(self, N):
        self.values = np.zeros((N,N,N,N,5))
        self.counts = np.zeros((N,N,N,N,5))
        
    def newstate(self, hider_loc, seeker_loc, updatevisits = True):
        visits = np.sum(self.counts[hider_loc[0], hider_loc[1], seeker_loc[0], seeker_loc[1], :])
        if visits > 0:
            return False
        else:
            if updatevisits:
                self.counts[hider_loc[0], hider_loc[1], seeker_loc[0], seeker_loc[1], :] +=1
            return True


class FullyObservableHideandSeek(object):
    """
    Last dimension of p1_dist corresponds to North East South West Stay
    """
    def __init__(self, hider, seeker, N, gamma=.5, c=2):
        self.N = N
        self.hider = hider
        self.seeker=seeker
        self.hider.create_value_and_counts(self.N)
        self.seeker.create_value_and_counts(self.N)
        self.event_pdf = [self.seeker.rate/float(self.seeker.rate +self.hider.rate), 
                          self.hider.rate/float(self.seeker.rate +self.hider.rate)]
        self.gamma = gamma
        self.c = c
        self.calls = 0
        self.optactions = []
        
    def do_maxn(self, numreps, resetstate_hider=None, resetstate_seeker = [0,0]):
        if resetstate_hider is None:
            resetstate_hider = [self.N-1, self.N-1]
        self.res = []
        self.time = []
        for rep in range(numreps):
            self.maxn_simulate()
            self.res.append(max(self.seeker.values[self.N-1, self.N-1, 0,0,:]))
            self.hider.loc = [self.N-1, self.N -1]
            self.seeker.loc = [0,0]
            self.time.append(time.time())
            self.optactions.append(np.argmax(self.seeker.values[self.N-1, self.N-1, 0,0,:]))
            
        return None
    
    def maxn_simulate(self,  t=0, game_over=False, player=None, nested_mcts=False):
        """
        Player is a player instance
        """
        q=np.array([0,0]) # Seeker and hider
        if game_over:
            return np.array([0,0])
        act_player = np.random.choice([self.seeker, self.hider], p = self.event_pdf)
        t += np.random.exponential(1/np.sum(np.asarray(self.event_pdf)))
        if act_player.newstate(self.hider.loc, self.seeker.loc):
            return self.rollout(act_player, t)
        move_row_val = act_player.values[self.hider.loc[0], self.hider.loc[1], self.seeker.loc[0], self.seeker.loc[1], :]
        move_row_cts = act_player.counts[self.hider.loc[0], self.hider.loc[1], self.seeker.loc[0], self.seeker.loc[1], :]
        move_ix = np.argmax(move_row_val + self.c *(np.log(np.sum(move_row_cts)) / np.float_(move_row_cts))**.5)
        move = ['N', 'E', 'S', 'W', 'P'][move_ix]
        self.update_state(act_player, move)
        game_over = self.hider.loc == self.seeker.loc
        move_row_cts[move_ix] += 1
        r = self.reward() + np.exp(-self.gamma*t)* self.maxn_simulate(t, game_over)
        if act_player.name=='seeker':
            move_row_val[move_ix] += (r[0] - move_row_val[move_ix])/move_row_cts[move_ix]
        else: 
            move_row_val[move_ix] += (r[1] - move_row_val[move_ix])/move_row_cts[move_ix]
        return r
        
    def rollout(self, player, t):
        is_valid = False
        while not is_valid:
            move = np.random.choice(('N', 'E', 'S', 'W', 'P')) #P for pass aka do notihng
            is_valid = self.eligible_move(player.loc, move)
        self.update_state(player, move)
        game_over = self.hider.loc == self.seeker.loc
        r = self.reward()
        return r + np.exp(-self.gamma * t) * self.maxn_simulate(game_over=game_over, t=t)
        
    def update_state(self, player, move):
        if (move == 'N') and (player.loc[1] > 0):
            player.loc[1] -=1
        if (move == 'S') and (player.loc[1] < self.N-1):
            player.loc[1] +=1
        if (move == 'E') and (player.loc[0] < self.N-1):
            player.loc[0] +=1
        if (move == 'W') and (player.loc[0] > 0):
            player.loc[0] -=1
    
    def reward(self):
        """
        Assign large negative rewards for going off grid
        """
        if self.seeker.loc == self.hider.loc:
            return np.array([1, -1])
        #elif self.seeker.loc ==[3,2]:
        #    return np.array([-10, 0])
        #elif self.seeker.loc==[2,2]:
        #    return np.array([-10, 0])
        else:
            return np.array([0,0])
        
    def eligible_move(self, loc, move):
        if (loc[0]==0) and (move=='W'):
            return False
        elif (loc[0]==self.N-1) and (move =='E'):
            return False
        elif (loc[1] ==0) and (move =='N'):
            return False
        elif (loc[1]==self.N-1) and (move =='S'):
            return False
        else:
            return True
        
    def do_nested_mcts(self, player, hiderstate, seekerstate, numsims = 1, gameover=False, initial=False, t=0):
        """
        Finds the optimal action for a player
        whose turn it is to move.  State is given by current
        state of hider and seeker
        """
        #print 'entering'
        if seekerstate == [0,0]:
            self.calls += 1
        if initial:
            self.res = []
            self.time = []
        if gameover:
            return 'P'
        self.hider.loc=copy.copy(hiderstate)
        self.seeker.loc=copy.copy(seekerstate)
        for j in range(numsims):
            self.nested_simulate(self.hider.loc, self.seeker.loc, player, t=t ,numsims=numsims)
            self.hider.loc = copy.copy(hiderstate)
            self.seeker.loc = copy.copy(seekerstate)
            self.res.append(max(self.seeker.values[self.N-1, self.N-1, 0,0,:]))
            self.time.append(time.time())
            self.optactions.append(np.argmax(self.seeker.values[self.N-1, self.N-1, 0,0,:]))

        ix = np.argmax(player.values[hiderstate[0], hiderstate[1],seekerstate[0], seekerstate[1], :])
        return ['N', 'E', 'S', 'W', 'P'][ix]
    
    def nested_simulate(self, hiderstate, seekerstate, player, gameover=False, t=0, numsims=1):
        act_player = player
        starting = copy.copy(seekerstate)
        if act_player.newstate(self.hider.loc, self.seeker.loc):
            move = self.nested_rollout(act_player)
            move_row_val = act_player.values[self.hider.loc[0], self.hider.loc[1], self.seeker.loc[0], self.seeker.loc[1], :]
            move_row_cts = act_player.counts[self.hider.loc[0], self.hider.loc[1], self.seeker.loc[0], self.seeker.loc[1], :]
            move_ix = ['N', 'E', 'S', 'W', 'P'].index(move)
        else:
            move_row_val = act_player.values[self.hider.loc[0], self.hider.loc[1], self.seeker.loc[0], self.seeker.loc[1], :]
            move_row_cts = act_player.counts[self.hider.loc[0], self.hider.loc[1], self.seeker.loc[0], self.seeker.loc[1], :]
            move_ix = np.argmax(move_row_val + self.c *(np.log(np.sum(move_row_cts)) / np.float_(move_row_cts))**.5)
            move = ['N', 'E', 'S', 'W', 'P'][move_ix]
        self.update_state(act_player, move)
        gameover = self.hider.loc == self.seeker.loc
        move_row_cts[move_ix] += 1
        if act_player.name=='seeker':
            r= self.reward()[0] 
        else:
            r=self.reward()[1]
        while not gameover:
            newplayer = np.random.choice([self.seeker, self.hider], p = self.event_pdf)
            t += np.random.exponential(1/np.sum(np.asarray(self.event_pdf)))
            move = self.do_nested_mcts(newplayer, copy.copy(self.hider.loc), copy.copy(self.seeker.loc), 
                                       gameover=gameover, numsims=numsims)
            self.update_state(newplayer, move)
            gameover = self.hider.loc == self.seeker.loc
            #move_row_cts[move_ix] += 1
            if act_player.name=='seeker':
                r += np.exp(-self.gamma *t) * self.reward()[0]
                #print r, self.seeker.loc
                #r += self.reward()[0] 
            else: 
                r += self.reward()[1]
        move_row_val[move_ix] += (r - move_row_val[move_ix])/float(move_row_cts[move_ix])
        
    def nested_rollout(self, player):
        is_valid = False
        while not is_valid:
            move = np.random.choice(('N', 'E', 'S', 'W', 'P')) #P for pass aka do notihng
            is_valid = self.eligible_move(player.loc, move)
        return move

Experiments 2.2 -Larger Grid¶

Nested MCTS, in it's simplest form cannot handle a 10X10 grid! The reason is that the MCTS runs until the end of the game, the only time rewards are accumulated. However, since it does not know where the reward cell is, it basically moves randomly until it finds the cell. Everytime there is a move, however it enters another nested MCTS.

General Comments¶

  • Nested MCTS is "less controllable"

    • Suppose we want to do MCTS for player $j$ in state $s$. Suppse we specify the outer loop to do UCT 10 times
    • However, the number of times we will actually do UCT from state $s$ for player $j$ is 10 times the number of times we revisit state $j$. This can lead to a lot of unnecessary looping.
    • Of course we can hand code in rules (for example, if a state has been visited more than a certain number of times, avoid doing MCTS and simply choose the action with the highest utility.
    • However, thos will not work in aprtially observable games as the underlying state of the world might be different

  • Nested MCTS grows multiplicatively with the depth, UCT grows additively. In other words, increasing the depth by 1 in MAX-N only adds one extra call to UCT. Increasing the MCTS depth by 1 adds another call to MCTS, which is results in numerous calls to UCT.

    • Nested MCTS had trouble with relatively small depths.
    • Nested MCTS will not be able to handle scenarios where substantial rewards are accumulated relatively far in the future.

  • Nested MCTS will be inefficient for determining the optimal move that seeds the MCTS. For each move that it tries, nested MCTS will have a more accurate expected utility for each possible action, however it will waste a lot of time on actions that do not yield the highest expected utility. In RL language, this is making the explore much more expensive when the unit of account is exploiting.

  • Other nested MCTS questions:

    • How do we set the depth for player's "down the line." Do we linearly decrease the depth? and if so, why?
