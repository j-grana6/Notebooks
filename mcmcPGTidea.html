MCMC "Intelligence"¶

The gist of the idea is that we are looking at the problem the wrong way. What we did was define a measure (intelligence) and then saught an algorithm that could sample efficiently from intelligences. Unfortunately, this was difficult because algorithms like MCMC need to satisfy detail balance (and some other properties). So, we were stuck looking for a smart way to sample.

Here, I take the opposite approach. Instead of deriving a quantity in which to base the probabilities, I first defined an algorithm (namely MCMC) and then defined an accept/reject criteria (ratio of probabilities) that fit in the algorithm and also "made sense" in the game theoretic way (i.e. it is close to intelligence). Let's take a look...

Here I only present the procedure/idea/ for games of imperfect information

Instead of defining probabilities of the PGT posterior, we define odds. We still want to capture some of the essence of intelligence so I chose to define odds as follows:

Ina two player game, if $s_1$ and $s_2$ are strategy profiles for player 1 and player 2. Then the PGT posterior odds of the profile $(s_1, s_2)$ to some alternative strategy profile $(s_1', s_2')$ is given by:

\begin{align} \frac{P(s_1, s_2)}{P(s_1', s_2')} \propto \frac{exp(\alpha EU_1(s_1, s_2))}{exp(\alpha EU_1(s_1', s_2))} \times \frac{ exp(\alpha EU_2(s_1, s_2))}{exp(\alpha EU_2(s_1, s_2'))}\times\frac{exp(\alpha EU_1(s_1, s_2'))}{exp(\alpha EU_1(s_1', s_2'))} \times \frac{exp(\alpha EU_2(s_1', s_2))}{exp(\alpha EU_2(s_1', s_2'))} \\ %&\propto \frac{exp\bigg(\alpha\big(EU_1(s_1, s_2) + EU_2(s_1, s_2) +EU_1(s_1, s_2') + EU_2(s_1', s_2)\big)\bigg)}%%{exp\bigg[\alpha\bigg(EU_1(s_1', s_2) + EU_2(s_1, s_2') + EU_1(s_1', s_2')+EU_2(s_1', s_2')\big)\bigg) } \propto exp\bigg(\alpha\big(EU_1(s_1, s_2) + EU_2(s_1, s_2) +EU_1(s_1, s_2') + EU_2(s_1', s_2) - EU_1(s_1', s_2) - EU_2(s_1, s_2') - EU_1(s_1', s_2')-EU_2(s_1', s_2')\big)\bigg) \end{align}

where $EU_i$, $i=1,2$ is the expected utility of player $i$ under the profiles in the argument of the function.

The tie this into the idea of intelligence, consider the first two terms. Those are minimal computation of the intelligence of $(s_1, s_2)$. The second term can be considered the multiplicative inverse of a small computation of the intelligence of $(s_1', s_2')$. The prodct of the exponential of all four terms is a measure of how intelligent $(s_1, s_2)$ is compared to $(s_1', s_2')$.

Another way to think about it is that the first term captures the idea of intelligence while the second term captures the idea of dominance. In other words, higher probability is assigned to profile where the strategies do well against each other but also do well against some of the opponents other strategies.

The $\alpha$ parameter controls the responsiveness of the probability to differences in intelligences. Also the need for exponentiating is give a monotone mapping of expected utilities onto $\mathbf{R}^+$ while also preserving 1) below.

The following properties hold :

  1. \begin{equation} \frac{P(s_1, s_2)}{P(s_1', s_2')} = \frac{1}{\frac{P(s_1', s_2')}{P(s_1, s_2)}} \end{equation}

  2. Any strategy profile where players are both playing dominant strategies receives the highest probability.

  3. All Nash equilibrium profiles do not receive the same weight. Instead, the weight they are assigned depends on how they perform against non-equilibrium strategies.

So why did I do this? If it isn't already clear, if we define the PGT distribution in terms of odds instead of probabilities, we can do MCMC by sampling strategy profiles uniformly and then accepting/rejecting based on the odds ratio. Note that getting a sample of the PGT distribution only requires evaluating the expected utility of 4 strategy profiles. Also note that the only way this grows is with the number of players.

def EU_perfect_info(p1strat, p2strat, utils=utils, p1top2map=mapdict):
    """ 
    Computes the expected utility of player 1 and player 2.
    
    Nature node is hard coded to p=.5 for both outcomes.
    
    Setting trembleprobs to 0 gives standard intelligence
    """
    highukey = "H"+p1strat[0]+p2strat[mapdict[p1strat[0]]]
    lowukey =  "L"+p1strat[1]+p2strat[3 + mapdict[p1strat[1]]]

def EU_with_trembles_perfect_info(p1strat, p2strat, utils=utils, p1top2map=mapdict, p1trembleprob=.2, 
                     p2trembleprob=.2, p1tremblestrats=perfect_strat_1, p2tremblestrats=perfect_strat_2):
    """ 
    Computes the expected utility of player 1 and player 2.
    
    Nature node is hard coded to p=.5 for both outcomes.
    
    Setting trembleprobs to 0 gives standard intelligence
    """
    highukey = "H"+p1strat[0]+p2strat[mapdict[p1strat[0]]]
    lowukey =  "L"+p1strat[1]+p2strat[3 + mapdict[p1strat[1]]]
    p1eunotremb = (1-p2trembleprob) * (.5 * utils[highukey][0] + .5 * utils[lowukey][0])
    p2eunotremb = (1-p1trembleprob) * (.5 * utils[highukey][1] + .5 * utils[lowukey][1])
    if p1trembleprob + p2trembleprob ==0:
        return p1eunotremb, p2eunotremb
    ### Get the utility that is associated with p2 trembling
    p1epovern = p1trembleprob/float(len(p1tremblestrats) - 1)
    p2epovern = p2trembleprob/float(len(p2tremblestrats) - 1)
    p1tremblingeu = 0 # Utility of p1 when p2 trembles
    p2tremblingeu = 0 # Utility of p2 when p1 trembles
    for s in p2tremblestrats:
        if s != p2strat:
            highukey = "H"+p1strat[0]+s[mapdict[p1strat[0]]]
            lowukey =  "L"+p1strat[1]+s[3 + mapdict[p1strat[1]]]
            p1tremblingeu += p2epovern * (.5 * utils[highukey][0] + .5 * utils[lowukey][0])
    for s in p1tremblestrats:
        if s != p1strat:
            highukey = "H"+s[0]+p2strat[mapdict[s[0]]]
            lowukey =  "L"+s[1]+p2strat[3 + mapdict[s[1]]]
            p2tremblingeu += p1epovern * (.5 * utils[highukey][1] + .5 * utils[lowukey][1])
    return p1eunotremb + p1tremblingeu, p2eunotremb + p2tremblingeu

def get_trembling_intelligence_perfect_info(profile, p1trembleprob=.2, p2trembleprob=.2):
    """
    Get's the trembling intelligence of a profile under trembling
    """
    p1strat = profile[0]
    p2strat = profile[1]
    util = EU_with_trembles_perfect_info(p1strat, p2strat, p1trembleprob=p1trembleprob, p2trembleprob=p2trembleprob)
    #print util
    p1_numalt = float(len(perfect_strat_1))
    p2_numalt = float(len(perfect_strat_2))
    num_no_betterp1 = 0
    num_no_betterp2 = 0
    for s in perfect_strat_1:
        if EU_with_trembles_perfect_info(s, p2strat, p1trembleprob=p1trembleprob, p2trembleprob=p2trembleprob)[0] <= util[0]:
            num_no_betterp1 +=1
    for s in perfect_strat_2:
        if EU_with_trembles_perfect_info(p1strat, s, p1trembleprob=p1trembleprob, p2trembleprob=p2trembleprob)[1] <= util[1]:
            num_no_betterp2 +=1
    return (num_no_betterp1-1)/(p1_numalt - 1) * (num_no_betterp2 - 1)/(p2_numalt -1)

class TremblingIntelResults(object):
    def __init__(self, res, stratprofiles = perfect_stratprofiles, perfect_strat_1=perfect_strat_1,
                 perfect_strat_2=perfect_strat_2):
        self.res = np.asarray(res)
        self.stratprofiles = stratprofiles
        self.sorted_ix = np.argsort(self.res)[::-1]
        self.perfect_strat_1 = perfect_strat_1
        self.perfect_strat_2 = perfect_strat_2
        
    def make_hist(self):
        plt.hist(self.res)
    
    def EU_perfect_info(self, p1strat, p2strat, utils=utils, p1top2map=mapdict):
        """
        No trembling EU.  USed to get Nash-eq
        """
        highukey = "H"+p1strat[0]+p2strat[mapdict[p1strat[0]]]
        lowukey =  "L"+p1strat[1]+p2strat[3 + mapdict[p1strat[1]]]
        p1eu = .5 * utils[highukey][0] + .5 * utils[lowukey][0]
        p2eu = .5 * utils[highukey][1] + .5 * utils[lowukey][1]
        return p1eu, p2eu
    
    def get_perfect_intelligence(profile):
        """
        Get's the non-trembling intelligences"""
        p1strat = profile[0]
        p2strat = profile[1]
        util = EU_perfect_info(p1strat, p2strat)
        #print util
        p1_numalt = float(len(self.perfect_strat_1))
        p2_numalt = float(len(self.perfect_strat_2))
        num_no_betterp1 = 0
        num_no_betterp2 = 0
        for s in self.perfect_strat_1:
            if EU_perfect_info(s, p2strat)[0] <= util[0]:
                num_no_betterp1 +=1
        for s in self.perfect_strat_2:
            if EU_perfect_info(p1strat, s)[1] <= util[1]:
                num_no_betterp2 +=1
        return (num_no_betterp1-1)/(p1_numalt - 1) * (num_no_betterp2 - 1)/(p2_numalt -1)
    
    def get_perfect_info_dist():
        intels_perfect = []
        for p in self.stratprofiles:
            intels_perfect.append(self.get_perfect_intelligence(p))
        return intels_perfect
    
    def eu_dist(self, power = 1, binwidth=10):
        total_prob = np.sum(self.res**power)
        probs = self.res**power/float(total_prob)
        eus = []
        for s in self.stratprofiles:
            eus.append(self.EU_perfect_info(s[0], s[1]))
        p1u = defaultdict(int)
        p2u = defaultdict(int)
        for ix, atom in enumerate(eus):
            p1u[int(atom[0]/binwidth) * binwidth ] += probs[ix]
            p2u[int(atom[1]/binwidth) * binwidth ] += probs[ix]
        return p1u, p2u
    
    def eu_hist(self, power=1, binwidth=10):
        eu_dicts = self.eu_dist(power=power, binwidth=binwidth)
        eu_subgame_perfect = [(625+250)/2. , (312. + 125.)/2.]
        eu_dicts_exact = self.eu_dist(power=power, binwidth=1)
        p1eeu = 0
        p2eeu = 0
        for k, v in eu_dicts_exact[0].iteritems():
            p1eeu += k * v
        for k, v in eu_dicts_exact[1].iteritems():
            p2eeu += k * v
        fig, [ax1, ax2] = plt.subplots(2)
        ax1.bar(eu_dicts[0].keys(), eu_dicts[0].values(), width=binwidth, color='r', alpha=.35)
        ax1.set_xlabel("Leader\'s expected utility")
        ax1.set_ylabel("Probability")
        ax1.vlines(eu_subgame_perfect[0], 0, .2)
        ax1.vlines(p1eeu, 0, .2, color='green')
        ax2.bar(eu_dicts[1].keys(), eu_dicts[1].values(), width=binwidth, color='r', alpha=.25)
        ax2.set_xlabel("Followers\'s expected utility")
        ax2.set_ylabel("Probability")
        ax2.vlines(eu_subgame_perfect[1], 0, .2)
        ax2.vlines(p2eeu, 0, .2, color='green')
        plt.tight_layout()
        fig.text(1, .5, 'The green vertical line is the expected utility of the intelligence profile. \n' + \
                        'The black line is the SPNE expected utility')
        return fig, ax1, ax2
    
    def eu_cdf(self, power=1):
        p1_dist, p2_dist = self.eu_dist( power = power)
        p1k = p1_dist.keys()
        p1v = p1_dist.values()
        p1_srtd = np.argsort(p1k)
        cumprob_p1 = np.cumsum(np.asarray(p1v)[p1_srtd])
        p2k = p2_dist.keys()
        p2v = p2_dist.values()
        p2_srtd = np.argsort(p2k)
        cumprob_p2 = np.cumsum(np.asarray(p2v)[p2_srtd])
        fig, [ax1, ax2] = plt.subplots(2)
        ax1.plot(np.asarray(p1k)[p1_srtd], cumprob_p1)
        ax1.set_xlabel("CDF of Leader\'s Utility")
        ax2.plot(np.asarray(p2k)[p2_srtd], cumprob_p2)
        ax2.set_xlabel("CDF of Followers\'s Utility")
        plt.tight_layout()
        return fig, ax1, ax2
